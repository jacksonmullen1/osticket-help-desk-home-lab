# Mapped Network Drives

## Purpose

This article documents the troubleshooting process for inaccessible mapped network drives.

---

## Symptoms

Users may report:

* Network drive unavailable
* Red X on mapped drive
* Windows cannot reconnect to the network drive
* Access denied to shared folders

---

## Troubleshooting Steps

1. Verify network connectivity.
2. Confirm the file server is online.
3. Verify user authentication.
4. Open File Explorer and remove the disconnected drive.
5. Remap the network drive using the correct shared folder path.
6. Confirm the drive reconnects successfully.
7. Verify access to files and folders.

---

## Resolution

The network drive was remapped successfully, and user access to the shared folder was restored.

---

## Notes

If multiple users experience the same issue, investigate file server availability before troubleshooting individual workstations.
